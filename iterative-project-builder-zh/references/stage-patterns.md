## RAG 系统模式

检索增强生成系统的 7 阶段划分。

| 天数 | 主题 | 核心功能 |
|------|------|----------|
| Day 1 | 最小化 RAG | 上传 → 分块 → 存储 → 检索 → 生成 |
| Day 2 | 数据预处理 | 多格式解析、智能分块、元数据 |
| Day 3 | 检索优化 | 混合检索（向量 + BM25）、重排序 |
| Day 4 | 生成增强 | 流式输出、引用溯源、防幻觉 |
| Day 5 | 评估监控 | RAGAS 指标、请求追踪 |
| Day 6 | 安全治理 | JWT 认证、ACL 权限、审计日志 |
| Day 7 | 生产就绪 | 缓存、指标、Docker 部署 |

**关键递进**：
- 核心流程 → 数据质量 → 检索质量 → 输出质量 → 评估 → 安全 → 部署

## 通用原则

### MVP 规则
Day 1 必须展示完整的核心价值：
- 用户可以完成主要任务
- 所有层协同工作（前端 + 后端 + 数据）
- 不是原型 - 生产级代码

### 独立性规则
每个阶段必须：
- 在自己的目录中自包含
- 无需其他阶段即可运行
- 完整的文档

### 增量规则
每天添加一个主要能力：
- 不是多个不相关功能
- 基于前一天的基础构建
- 明确的主题和目标

### 文档规则
每天包括：
- README（中英文）
- CHANGES.md 解释与前一天的差异
- 更新的规划文件

### 命名约定
```
day1/     # 最小化/MVP
day2/     # 增强名称
day3/     # 增强名称
...
dayN/     # 生产/完整
```
